slackware-7.1 install

　slackware-7.1を入手し，自作ＡＴ互換機にinstallします。ＨＤとＣＤ－ＲＯＭは， ＩＤＥ接続されているという前提で書いています。

（インストール後，しばらくたってから書いてますので，記憶があいまいです）


  1. 必要ファイルの準備

  2. ＣＤ－ＲＯＭをつくる

  3. boot disk の作成

  4. root disk の作成

  5. ハードディスクのパーテーション作成

  6. install

  7. 起動

  1. 必要ファイルの準備
    ftp.slackware.com/pub/slackware/slackware-7.1/配下の
    • bootdsks.144/
    • rootdsks/
    • slakware/
    の中から，必要な物を頂く。必要なファイルは，ハードディスクと ＣＤ－ＲＯＭがIDE接続なら，
    • bootdsks.144/から，
      • RAWRITE13.EXE
      • bare.i
    • rootdsks/から，
      • color.zg
    くらいが必要。
    • slakware/配下は，
      • wget -m -np -b とかやって，かき集める。

    雑誌の付録等で入手出来れば吉なんでしょうけど・・・

      　なぜ，iso形式のＣＤ－ＲＯＭイメージをgetしなかったかというと・・・
      　本所からInternetへは，青葉山Proxy server経由だが， １本のファイルのダウンロードに３時間２０分以上かかる物は ダウンロード出来なかった（多分青葉山proxyの設定）。 wget -c も，不可だった。
      　よって，*.isoをgetし，ＣＤ－ＲＯＭに焼くのは断念した。
      （私が関わっている場所の環境の特殊な環境でのお話でした ^o^）。

    　slackwareのメーリングリスト の中から，必要と思われるものにsubscribeしておくと，情報がもらえる。


  2. ＣＲ－ＲＯＭをつくる
    　ダウンロードしたファイルをＣＤ－ＲＯＭに焼いておく。
    　この時，ＣＤ－ＲＯＭのルートディレクトリにslakware/（へのシンボリックリンク） を置いておくのがコツ。 (こうしておくと，installの時に，＜ＣＤ－ＲＯＭからのinstall＞を選んだだけで， 上手く行くようになる)

  3. bootdiskの作成
    　RAWRITE13.EXEを使って，bare.iをフロッピーに書き込む。
    　Windowsのdos窓からだと上手く行かないかも知れない。


  4. rootdiskの作成
    　RAWRITE13.EXEを使って，color.gzをフロッピーに書き込む。
    　Windowsのdos窓からだと上手く行かないかも知れない。


  5. ハードディスクのパーテーション作成
    　バックアップツールを使ってバックアップする時の便を考え，次の様なパーテーション 構成にした。
    • FAT16
    • FAT16
    • Linux native
    • Linux swap

    手順
    1. パーテーションを切る前に，ハードディスク内のデータがbackupされているか確認する。
    2. bootdiskでマシン起動。
    3. 途中で，指示に従い，rootdiskをドライブに入れ，起動処理を続けさせる。
    4. 起動したら，rootでログイン
    5. cfdiskを実行
      cfdiskで次のように設定する

      Heads: 255 Sectors per Track: 63 Cylinders: 1232
      　Name　Flags　Part Type　FS Type　[Label]　Size(MB)
      　hda1　Boot　Primary　FAT16　[]　2146.80
      　hda5　Logical　FAT16　[NO NAME]　2146.80
      　hda3　Primary　Linux ext2　5000.98
      　hda4　Primary　Linux swap　 838.98

    6. 結果を書き込み，cfdiskを終了。bootdiskを挿入し，Ctrl+Alt+Ddeleteで再起動。


  6. install
    　rootでログインし，setupを実行するとメニューが表示されるが，その中の KEYMAPを選択すると，以後，TARGET SOURCE SELECT INSTALL CONFIGUREの中身が 順次実行される。
    • KEYMAP は，とりあえずus.mapにしておく。 （日本語106キーボードは後で設定する）。
    • TARGETでは，/ としてmountするパーテーション，swapとしてmountするパーテーションが聞かれる。 （cfdiskでFS type を正しく設定していれば，それを探して，表示してくれる。）
    • SOURCEでは，ＣＤ－ＲＯＭからのinstallを選択。
    • SELECTでは，お好みに応じて（全てを）選択。
    • INSTALLでは，先人達に感謝しながら画面を眺める。
    • CONFIGUREでは，ネットワーク関連と，タイムゾーン以外は，お好みに応じて。
      　ネームサーバは，とりあえず設定しておき，後できちんと設定しなおす。
      　ここで，もし，＠マークを入力したくなったら，Ｐの右隣のキーの代りに， SHIFT+2(")を押す。
      　ネットワーク関連は，ＮＩＣを自動で認識させ，設定ファイルを作らせておくと， 後の作業で，見通しが良い。
        　ISAタイプのNE2000互換カードが刺さった状態で自動認識させるとハングアップする らしいが，PCIタイプの他のカードが刺さっていると，そちらを先に（それだけを） 認識してくれる様だ。
      　liloディスクは作っておく。
      　lilo configurationは，自動で作り，実行しておく。
      　rootのパスワードについて。
      　たしか，この辺で，rootのパスワードを決めるように言われる。
      この時，＜Num Lockの状態の確認＞と，＜キーボードのマッピングが狂っている＞ ということが理解出来ていないと，キーボードの設定をした後で一瞬焦る（かも知れない）。
      　何を言っているのかが解らなければ，ここではアルファベットのみの （記号・数字を使わない）パスワードを設定しておく。
      　ただし，この様なパスワードは好ましくないので， キーボードの設定を日本語106キーにした後で，passwdコマンドを実行し，パスワードを 設定し直す。

  7. 起動
    全てが終わったら，メッセージに従い，EXIT し，reboot。
    上手く行くと起動する。

終わったら基本的な設定等に進む。
